# hub-zh-c

一个用于归档和发布独立 HTML 页面的通用仓库。本项目不针对任何特定域名、网站或服务，仅作为静态 HTML 资源的集中存储与展示空间。

## 项目简介

本仓库旨在收集、整理并托管一系列功能独立的 HTML 页面。这些页面可能包含工具、演示、信息展示或其他用途，彼此之间没有强关联，每个页面均可独立访问和使用。

## 目录说明

```
.
├── index.html          # 仓库主页 / 导航页
├── pages/              # 存放各独立 HTML 页面的子目录
│   ├── example1.html
│   ├── example2.html
│   └── ...
└── assets/             # 共用的静态资源（CSS、JS、图片等）
    ├── css/
    ├── js/
    └── images/
```

- `pages/`：每个 HTML 页面作为一个独立文件存放，文件名建议使用英文小写、短横线分隔。
- `assets/`：存放页面共用的样式、脚本或媒体资源，避免重复引用。

## 页面归档说明

- 每个页面应当功能完整、自包含，尽量不依赖外部资源（或依赖集中存放于 `assets/`）。
- 页面之间不应存在硬编码的相互引用，除非明确说明。
- 归档的页面可能来自不同时期、不同用途，质量与风格可能不统一，但不影响其独立运行。

## 使用方式

1. 直接访问 GitHub Pages 部署的站点（如已启用），或通过本地克隆后使用浏览器打开 `index.html`。
2. 浏览 `pages/` 目录下的文件列表，点击对应链接即可查看独立页面。
